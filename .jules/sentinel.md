## 2025-02-27 - Insecure Deserialization and ACE in PyTorch/Transformers
**Vulnerability:** Found `torch.load` being used without `weights_only=True` and HuggingFace Auto classes (e.g., `AutoTokenizer`, `AutoModelForCausalLM`) hardcoded with `trust_remote_code=True`.
**Learning:** PyTorch's `torch.load` uses `pickle` under the hood, which is vulnerable to Arbitrary Code Execution (ACE) / Insecure Deserialization if loading untrusted files. Similarly, HuggingFace models can include malicious Python code that runs upon loading if `trust_remote_code=True` is enabled. Since this CLI tool loads potentially untrusted `.pt` files and models, this poses a severe security risk.
**Prevention:** Always use `weights_only=True` when calling `torch.load` for loading model weights or simple configuration data. Default `trust_remote_code` to `False` when loading HuggingFace models, requiring users to explicitly opt-in via a CLI flag (e.g., `--trust-remote-code`) if they intend to trust the remote code.
